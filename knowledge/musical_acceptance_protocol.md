# Musical Acceptance Protocol (MAP) — ENTROP

A milestone does not pass until it passes MAP. Technical correctness alone is not sufficient.

## Musical utility criteria — need 3/5 to pass
1. Intent readability — user hears what changed when moving a control
2. Performance stability — result is controllable for phrase playing
3. Mix plausibility — sound fits in a composition after standard treatment
4. Expressive gradient — audible change across parameter travel, no dead zones
5. Recallability — user can return to the sound via preset or archetype

## Required listening tests per milestone

| Test | Pass Condition |
|---|---|
| TTUS | ≤60s experienced / ≤120s intermediate |
| Control gradient | Audible change every 10% increment |
| Archetype identity | Clear distinct character per archetype |
| Edit robustness | ≥7/10 edits ±20% from archetype center keep musicality |
| Long-form static | Audible evolution over 60s (DIFFU/BUBBLE) |
| Arrangement fit | Patch sits in minimal mix without rescue |
| MPE expression | Enhances rather than destabilizes |
